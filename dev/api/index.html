API Reference

Radio Types

Main.YaesuCAT.FT891 — Type
FT891{T <: AbstractTransport} <: AbstractYaesuRadio

Yaesu FT-891 HF/50 MHz transceiver. Parametric on transport type T for compile-time type stability.

Constructors

# Production — serial port (uses SerialTransport internally)
radio = FT891("/dev/ttyUSB0"; baudrate=9600, timeout_ms=200)

# Testing — null transport
radio = FT891(NullTransport())

Radio identification

The FT-891 returns "0670" for the ID command.

Connection

Main.YaesuCAT.connect! — Function
connect!(radio::AbstractYaesuRadio)

Open the serial transport to the radio. Must be called before any set! or read operations.

Main.YaesuCAT.disconnect! — Function
disconnect!(radio::AbstractYaesuRadio)

Close the serial transport and release the port.

High-Level API

Main.YaesuCAT.set! — Function
set!(radio, cmd, value)

Set a radio parameter. The command type and value type together determine the CAT string that gets sent.

set!(radio, FrequencyVFO{A}(), Hz(14_060_000))
set!(radio, Mode(), CW())
set!(radio, KeySpeed(), WPM(17))
set!(radio, CWKeying(), KeyerSlot(1))
Base.read — Function
read(radio, cmd) → value

Read a radio parameter. Returns a typed value (e.g., Hz, WPM, CW()).

freq = read(radio, FrequencyVFO{A}())  # → Hz(14060000)
wpm  = read(radio, KeySpeed())          # → WPM(17)
mode = read(radio, Mode())              # → CW()
id   = read(radio, Identification())    # → "0670"

Low-Level API

Main.YaesuCAT.send_cmd! — Function
send_cmd!(radio, cmd_str::AbstractString)

Send a raw CAT command string. Terminator is appended if missing. Prefer the typed set! / read API for safety.

Main.YaesuCAT.read_response — Function
read_response(radio; timeout_ms) → String

Read raw bytes from the radio until the terminator ; or timeout.

Main.YaesuCAT.query — Function
query(radio, cmd_str) → String

Send a raw command and return the stripped response.

Commands — Frequency & Mode

Main.YaesuCAT.FrequencyVFO — Type
FrequencyVFO{V <: AbstractVFO} <: SetReadCommand

VFO frequency command. Parametric on VFO tag for compile-time dispatch. CAT codes: FA (VFO-A), FB (VFO-B). Capabilities: Set/Read/Answer.

set!(radio, FrequencyVFO{A}(), Hz(14_250_000))
freq = read(radio, FrequencyVFO{A}())  # → Hz(14250000)
Main.YaesuCAT.Mode — Type
Mode <: SetReadCommand

Operating mode command. CAT code: MD. Capabilities: Set/Read/Answer. P1 is fixed to 0 for the FT-891 (main VFO).

set!(radio, Mode(), CW())
mode = read(radio, Mode())  # → CW()

Commands — CW Keyer

Main.YaesuCAT.KeySpeed — Type
KeySpeed <: SetReadCommand

CW keyer speed. CAT code: KS. Range: 4–60 WPM.

set!(radio, KeySpeed(), WPM(17))
wpm = read(radio, KeySpeed())  # → WPM(17)
Main.YaesuCAT.Keyer — Type
Keyer <: SetReadCommand

Internal electronic keyer enable/disable. CAT code: KR.

Main.YaesuCAT.KeyPitch — Type
KeyPitch <: SetReadCommand

CW sidetone pitch. CAT code: KP. Range: 300–1050 Hz, 10 Hz steps. Parameter: (hz - 300) ÷ 10, formatted as 2 digits.

Main.YaesuCAT.CWKeying — Type
CWKeying <: SetOnlyCommand

CW keyer memory / message playback trigger. CAT code: KY. Set only — the FT-891 does not support reading the keying state.

This command triggers playback of a pre-stored message. It does NOT accept arbitrary text. Use KeyerMemory to store messages first, or use RTSKeyer for arbitrary CW text.

set!(radio, CWKeying(), KeyerSlot(1))   # play keyer memory 1
set!(radio, CWKeying(), MessageSlot(3)) # play message keyer 3
kd-boss/CAT reference

In the C++ library, CWKeying::Set() maps the same way — it only triggers playback of stored messages. There is no Read() or Answer() for this command (column pattern O/X/X/X).

Main.YaesuCAT.KeyerMemory — Type
KeyerMemory <: SetReadCommand

CW keyer memory storage. CAT code: KM. Slots 1–5, up to 50 characters each. Set stores a message; Read retrieves the stored text.

set!(radio, KeyerMemory(1), "CQ CQ DE SA0KAM K")
msg = read(radio, KeyerMemory(1))  # → "CQ CQ DE SA0KAM K"

Commands — TX Control

Main.YaesuCAT.TXState — Type
TXState <: SetReadCommand

Transmit state control. CAT code: TX. Values: 0 = all off, 1 = CAT TX on, 2 = radio TX on.

CW Keying

Main.YaesuCAT.RTSKeyer — Type
RTSKeyer <: AbstractCWKeyer

CW keyer that toggles the RTS line on a serial port. Requires FT-891 menu 07-12 PC KEYING set to RTS.

The keyer uses the Standard COM port (the lower-numbered /dev/ttyUSB port), separate from the Enhanced port used for CAT commands.

keyer = RTSKeyer("/dev/ttyUSB1")
send_morse!(keyer, "CQ CQ DE SA0KAM K", WPM(17))
close(keyer)
Main.YaesuCAT.DTRKeyer — Type
DTRKeyer <: AbstractCWKeyer

CW keyer that toggles the DTR line. Same as RTSKeyer but for radios configured with 07-12 PC KEYING = DTR.

Main.YaesuCAT.send_morse! — Function
send_morse!(keyer::AbstractCWKeyer, text::AbstractString, speed::WPM)

Send arbitrary Morse code by toggling the keyer's serial control line.

Opens the serial port if not already open. The port is left open after sending so multiple messages can be sent without re-opening.

Arguments

  • keyer: An RTSKeyer or DTRKeyer connected to the Standard COM port
  • text: ASCII text to send as Morse code
  • speed: Keying speed as WPM

Timing

Uses PARIS standard timing: 1 unit = 1200/WPM milliseconds.

FT-891 Setup

  • Menu 07-12 PC KEYING: set to RTS (for RTSKeyer) or DTR (for DTRKeyer)
  • Mode must be CW or CW-R
  • Keyer type (menu 04-01) should be set to OFF for straight-key emulation

Example

keyer = RTSKeyer("/dev/ttyUSB1")
send_morse!(keyer, "CQ CQ DE SA0KAM SA0KAM K", WPM(17))
send_morse!(keyer, "5NN TU", WPM(17))
close(keyer)
send_morse!(keyer::AbstractCWKeyer, elements::Vector{MorseElement}, speed::WPM)

Send pre-computed Morse elements. Useful when you want to pre-process the text once and send it multiple times.

Main.YaesuCAT.text_to_morse — Function
text_to_morse(text::AbstractString) → Vector{MorseElement}

Convert a text string to a flat vector of MorseElement timing codes. Unknown characters are silently skipped.

elements = text_to_morse("CQ CQ")
# [DAH, ELEMENT_GAP, DIT, ELEMENT_GAP, DAH, ELEMENT_GAP, DIT,
#  CHAR_GAP,

#  WORD_GAP,
#  DAH, ELEMENT_GAP, DIT, ELEMENT_GAP, DAH, ELEMENT_GAP, DIT,
#  CHAR_GAP,
#  DAH, ELEMENT_GAP, DAH, ELEMENT_GAP, DIT, ELEMENT_GAP, DAH]

Protocol (Internal)

Main.YaesuCAT.encode_set — Function
encode_set(cmd, value) → String

Encode a Set command into a CAT protocol string (with terminator). Dispatches on (CommandType, ValueType) — fully resolved at compile time.

encode_set(FrequencyVFO{A}(), Hz(14_060_000))  # → "FA014060000;"
encode_set(Mode(), CW())                        # → "MD03;"
encode_set(KeySpeed(), WPM(17))                  # → "KS017;"
Main.YaesuCAT.encode_read — Function
encode_read(cmd) → String

Encode a Read command (request for current value).

encode_read(FrequencyVFO{A}())  # → "FA;"
encode_read(KeySpeed())          # → "KS;"
Main.YaesuCAT.decode_answer — Function
decode_answer(cmd, raw::AbstractString) → value

Decode an Answer string from the radio into the appropriate value type.

decode_answer(FrequencyVFO{A}(), "FA014060000")  # → Hz(14060000)
decode_answer(KeySpeed(), "KS017")                 # → WPM(17)
decode_answer(Mode(), "MD03")                      # → CW()

Transport (Internal)

Main.YaesuCAT.SerialTransport — Type
SerialTransport <: AbstractTransport

Production serial transport wrapping a LibSerialPort SerialPort. Stores the port name, baud rate, and timeout for connection management.

The FT-891 uses 8 data bits, no parity, 2 stop bits (8N2).

Main.YaesuCAT.NullTransport — Type
NullTransport <: AbstractTransport

A no-op transport for testing and dry-run scenarios. write_bytes records commands; read_until_terminator returns a configurable response.

t = NullTransport()
push!(t.responses, "FA014060000;")  # pre-load a mock response

Type Hierarchy

Main.YaesuCAT.AbstractRadio — Type
AbstractRadio

Root type for all radio transceivers. Subtypes must implement connect!, disconnect!, and the transport interface.

Main.YaesuCAT.AbstractYaesuRadio — Type
AbstractYaesuRadio <: AbstractRadio

Yaesu radios using the CAT command protocol (semicolon-terminated ASCII). All Yaesu CAT commands share the same framing: CMD[PARAMS];

Main.YaesuCAT.AbstractCommand — Type
AbstractCommand

Root of the command type tree. Every CAT command is a singleton (zero-size) type descending from one of the capability branches below.

Main.YaesuCAT.SetReadCommand — Type
SetReadCommand <: AbstractCommand

Commands that support both set! and read. Most common category. Corresponds to columns O/O/O in the Yaesu CAT command table.

Main.YaesuCAT.SetOnlyCommand — Type
SetOnlyCommand <: AbstractCommand

Commands that only support set! (no read/answer). Column pattern: O/X/X/X in the Yaesu CAT command table. Examples: BandDown, BandUp, CWKeying.

Main.YaesuCAT.ReadOnlyCommand — Type
ReadOnlyCommand <: AbstractCommand

Commands that only support read (with answer). Column pattern: X/O/O/X in the Yaesu CAT command table. Examples: Information, Busy.

Main.YaesuCAT.AbstractVFO — Type
AbstractVFO

Phantom type tag for VFO selection. Used as a type parameter to distinguish VFO-A vs VFO-B commands at compile time.

FrequencyVFO{A}()  # VFO-A frequency command
FrequencyVFO{B}()  # VFO-B frequency command — different type, different dispatch
Main.YaesuCAT.AbstractValue — Type
AbstractValue

Root for all parameter value types. Using dedicated types instead of raw Int/String gives us:

  • Type safety (can't pass Hz where WPM is expected)
  • Self-documenting call sites
  • Dispatch without runtime checks
Main.YaesuCAT.AbstractMode — Type
AbstractMode <: AbstractValue

Operating modes (CW, USB, LSB, FM, AM, DATA, RTTY variants). Each mode is a singleton type for zero-cost dispatch.

Main.YaesuCAT.AbstractBreakInMode — Type
AbstractBreakInMode <: AbstractValue

CW break-in modes: off, semi, full.

Main.YaesuCAT.AbstractAGCMode — Type
AbstractAGCMode <: AbstractValue

AGC function settings.

Main.YaesuCAT.AbstractMeterType — Type
AbstractMeterType <: AbstractValue

Meter display types (compression, ALC, power out, SWR, current).

Main.YaesuCAT.AbstractSwitch — Type
AbstractSwitch <: AbstractValue

Binary on/off toggle. Use On() or Off() — no booleans needed.

Main.YaesuCAT.AbstractBand — Type
AbstractBand <: AbstractValue

HF/VHF band selection values.

Main.YaesuCAT.AbstractTransport — Type
AbstractTransport

Abstraction for the serial communication layer. Implement write_bytes and read_until_terminator for new transports. The default SerialTransport wraps LibSerialPort.

Main.YaesuCAT.AbstractCWKeyer — Type
AbstractCWKeyer

Root type for CW keying backends. Subtypes control how Morse code is physically keyed:

  • RTSKeyer: toggles RTS line on the Standard COM port
  • DTRKeyer: toggles DTR line on the Standard COM port
